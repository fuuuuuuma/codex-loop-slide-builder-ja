# make-slides — Codex × ループでスライドを半自動生成

このスキルは「AIにゴールと合格条件だけ渡して、生成 → 検証 → 改善 を自分で回させる（ループエンジニアリング）」を、
スライド作成にそのまま適用するための手順書です。**実際に全140枚のセミナースライドを1セッションで仕上げた往復の型**を、誰でも再現できる形に落としています。

このフォルダ（`make-slides/`）を丸ごとエージェント（Codex / Claude Code）に渡し、「このスキルでスライドを作って」と言えば動きます。

## このスキルが作るもの

- **単一HTMLのスライド**（1280×720・ブラウザで開くだけ・配布も簡単）
- **1スライド1メッセージ**・**はみ出しゼロ**・**通し番号は自動**

## 同梱ファイル（役割分担＝AIが迷わない土台）

| ファイル | 役割 |
|---|---|
| `AGENTS.md` | AIが毎回守る社内ルール（禁止ワード・1スライド1メッセージ・数値は出典必須） |
| `DESIGN.md` | 配色・フォント・型の正典。`../design-systems/` から1つ選んでここにコピー |
| `outline.template.md` | 構成（章立て・見出し）のひな型 |
| `script.example.json` | 1スライド = 1ブロックのデータ例（**文章と見た目を分ける**） |
| `slides.template.html` | 空のスライド骨格。**番号・カウンターはJSが自動採番** |
| `check.mjs` | はみ出し検出（評価役＝ループの心臓） |
| `prompts.md` | Phase別のコピペ用プロンプト |

## 手順（このループを回す）

```
Phase 0 準備 … 伝えたい結論（コアメッセージ）を1行で決める          ← 人（監督）
Phase 1 骨子 … outline.md に章立て（見出しだけ）を作る              ← AI＋人
Phase 2 生成 … script.json に1枚ずつ本文（1スライド1メッセージ）    ← AI
              → slides.html に流し込む
Phase 3 改善 … 批評 → 改善を 3〜5周。死にスライド排除・ヘッダー統一  ← AI（ループ）
Phase 4 検証 … check.mjs で はみ出し0 まで直す。枚数・番号を整合     ← AI（評価役）
```

各Phaseのコピペ用プロンプトは `prompts.md` にあります。

## 守ること（`AGENTS.md` と共通）

1. **1スライド1メッセージ**。言いたいことを詰め込まない。
2. **文章（script.json）と見た目（slides.html）を分ける**。中身だけ直しても見た目が崩れない。
3. **数値・価格・日付は出典が取れなければ書かない**。確度マーク（✅公式 / 🔶一次情報 / ⚠️未確認）を各主張に付ける。
4. **誇張・断定保証をしない**（必ず／絶対／確実／神／ヤバい／裏技 等は使わない）。
5. **通し番号・枚数はハードコードしない**。スライドの数を数えて流し込む（`slides.template.html` のJSが自動でやる）。
6. **死にスライドは「説明が弱い方」を消す**（同じ主張の重複は、専門的すぎる版を消して分かりやすい版を残す）。
7. **参加・思考をうながす問いを 5〜8枚に1回**入れて、聞く側のリズムを作る。

## ループの肝（なぜ回るのか）

スライドは「良し悪しを**出力で○×が付けられる**」タスクです。

- 文字がはみ出していないか → `check.mjs` が高さを測って判定
- 1スライド1メッセージか → 見出し・本文量で判定
- 枚数・通し番号が合っているか → 数えれば分かる
- 死にスライドが無いか → 重複を検出

だから「**作る役（生成）と採点する役（check.mjs）を分け、両方OKだけ合格**」にすれば、AIが自分で「はみ出してる→直す」を回せます。

## 発展（凝るときだけ）

- **コメントアンカー編集**：スライドは `<!-- 番号 名前 -->` ＋ `<section>` の単位。番号でなく**コメント文字列をアンカー**に「削除／差し替え／直前挿入」すると、番号がズレても安全。
- **顔写真などの画像**：外部URL参照だと配布時にパス切れする。`curl` で取得 → 縮小 → **base64 で `data:` URIに埋め込み**、単一HTML自己完結を保つ。
- **役割分担**：見た目（配色・図解）はデザイン特化ツール、構造（文章・骨組み）はCodex、と分けると凝っても崩れにくい。
- **PPTX化**：完成スライドを2倍スクショ → `python-pptx` で 13.333×7.5inch に画像貼り（崩れゼロで配布）。
- **公開**：単一HTMLなので GitHub Pages 等にそのまま置ける。

## ツール別メモ

- **Claude Code**：このフォルダを `.claude/skills/make-slides/` に置くか、フォルダごと会話に渡す。
- **Codex**：`AGENTS.md` を運用ルールとして読み込ませ、`SKILL.md` の手順に沿って実行。`/goal` でゴール条件（はみ出し0・全◯枚）を渡すと自走しやすい。
