# prompts.md — そのままコピペで使うプロンプト集

`make-slides` の各Phaseで、エージェント（Codex / Claude Code）に投げる文です。
`◯◯`（テーマ）だけ書き換えて使ってください。

---

## 0. 最初の1投（土台を渡す）

```
このフォルダの make-slides スキルでスライドを作ります。
AGENTS.md のルールを必ず守ってください。
デザインは ../design-systems/ から「△△（テーマ名）」の DESIGN.md を
make-slides/DESIGN.md にコピーして使ってください。
テーマは「◯◯」。まずは Phase 1（章立て）からお願いします。
```

## Phase 1 — 骨子（構成だけ作る）

```
◯◯のスライドの章立てを作って。
- 各スライドは「見出し1行 ＋ 一言メモ」だけ
- 1スライド1メッセージ
- 全体で10〜12枚を目安に
- 5〜8枚に1回、見ている人に考えさせる問いを入れる
outline.md に書き出してください。
```

## Phase 2 — 生成（1枚ずつ本文）

```
outline.md の各スライドを、script.json に1ブロックずつ起こして。
- 文章（中身）だけを script.json に。見た目は触らない
- 1スライド1メッセージを厳守
- 数値・価格・日付は出典が取れないものは書かない（確度マークを付ける）
そのあと slides.html に流し込んでください。
```

## Phase 3 — 改善ループ（ここが“ループ”）

```
各スライドを self-review してください。
- 「無くても困らない／重複している」死にスライドを見つけて、統合または削除
- 重複は “説明が弱い方” を消す
- 見出しの形（ヘッダー）を全スライドで統一
合格と思えるまで、批評 → 改善を最大5周くり返してください。
```

## Phase 4 — 検証（評価役）

```
check.mjs を実行して、はみ出しが出たスライドを直してください。
はみ出しが 0 になるまで、検出 → 修正 を自動でくり返して。
最後に、通し番号・枚数・目次が一致しているかを確認してください
（数字はハードコードせず、スライド数を数えて流し込む）。
```

---

## 台本・記事からスライドにする場合

```
次の台本（または記事）を、make-slides スキルでスライドにして。
- まず「悩み → 結論 → 3本柱 → 使い方 → まとめ」のナラティブに再構成
- 1スライド1メッセージ
- 専門用語は日常語・比喩に開く（初心者基準）
台本：
<ここに台本・記事を貼る>
```

## よく効く追い込みプロンプト

```
- 「この1枚、言いたいことが2つ以上ある。1つに削って」
- 「この主張、出典は？ 無ければ確度マークを下げるか落として」
- 「全体を通しで見て、流れが切れている所・退屈になる所を3つ挙げて直して」
- 「このデッキの“掴み”（最初の3枚）を、見ている人の悩みから始まる形に」
```
