# AGENTS.md — スライド作成の社内ルール

このファイルは、AI（Codex / Claude Code）がスライドを作るときに**毎回守るルール**です。
`make-slides` スキルと一緒に読み込ませてください。Claude Code では `CLAUDE.md` としても使えます（`ln -s AGENTS.md CLAUDE.md` など）。

## 必ず守る

- **1スライド1メッセージ**。1枚に言いたいことは1つだけ。詰め込まない。
- **文章と見た目を分ける**。本文は `script.json` に、装飾・レイアウトは `slides.html` に。中身だけ直して見た目を壊さない。
- **数値・価格・日付・固有名詞は、出典が取れなければ書かない**。各主張に確度マークを付ける：
  - ✅ 公式確認済み ／ 🔶 一次情報あり ／ ⚠️ 未確認・推測
- **通し番号・枚数はハードコードしない**。スライド数を数えて流し込む（テンプレのJSが自動）。
- **死にスライドは「説明が弱い方」を消す**。同じ主張の重複は、専門的すぎる版を削り、分かりやすい版を残す。
- **参加・思考をうながす問いを 5〜8枚に1回**入れる（番号／○×／作業名ひとつで答えられる具体的な問い）。

## 禁止表現（誇張・断定保証）

次の言葉は使わない（内容で回収できない煽りは逆効果）：

```
必ず / 絶対 / 確実 / 裏技 / 神 / ヤバい / 衝撃 / 100%稼げる / 誰でも一瞬で
```

効果は「〜も目指せる」「〜しやすくなる」のように中立に書く。景品表示法・薬機法に触れる断定はしない。

## 口調・文体

- 初心者基準。専門用語を最初に出さず、日常語・比喩・「悩み→こう頼むだけ」で説明する。
- 開発者向けの重い話は「もう一歩進んだ使い方」に小さく退避し、本筋は軽く保つ。

## 出力の形

- スライドは**単一HTML**（1280×720）。外部依存を増やさない。画像は base64 で埋め込み、配布時のパス切れを防ぐ。
- 完成後は `check.mjs` で**はみ出しゼロ**を確認してから「完成」と言う（実機検証なしに完成と言わない）。
