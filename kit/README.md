# 🎁 スライド量産キット（そのまま渡せば動く）

このフォルダ（`kit/`）は、**受け取った人がそのまま使える実ファイル**の詰め合わせです。
読み物ではなく「渡す → 動く」道具一式。中身は2つ。

```
kit/
├── make-slides/      ← ① スライド作成スキル（手順＋雛形ファイル一式）
│   ├── SKILL.md            このスキルの本体（Codex / Claude Code が読む）
│   ├── AGENTS.md           守るルール（社内ルールブック）
│   ├── prompts.md          Phase別のコピペ用プロンプト
│   ├── outline.template.md 構成のひな型
│   ├── script.example.json 1スライド=1ブロックのデータ例
│   ├── slides.template.html 空のスライド骨格（番号は自動採番）
│   └── check.mjs           はみ出し自動検出（評価役）
└── design-systems/   ← ② スライド配色・型の正典 50種
    ├── INDEX.md            50種の一覧（ここから選ぶ）
    └── 01-….DESIGN.md 〜 50-….DESIGN.md
```

---

## いちばん簡単な使い方（3ステップ）

1. **デザインを選ぶ** — `design-systems/INDEX.md` を開き、好きなテーマ（例：`21-money-emerald`）を決める。
   その `.DESIGN.md` を `make-slides/DESIGN.md` という名前でコピーする。
2. **エージェントに渡す** — Codex か Claude Code を `make-slides/` フォルダで開き、こう言う：

   ```
   このフォルダの make-slides スキルでスライドを作って。
   AGENTS.md を必ず守って。デザインは DESIGN.md に従って。
   テーマは「◯◯」。まず章立て（Phase 1）から。
   ```

3. **ループを回す** — `prompts.md` の Phase 1 → 4 を順に投げるだけ。
   最後に `check.mjs` で「はみ出しゼロ」を確認したら完成。

> これだけで、選んだ配色に統一された・はみ出しゼロ・1スライド1メッセージのスライド（単一HTML）が出来ます。

---

## ① make-slides スキルとは

「AIにゴールと合格条件だけ渡して、生成 → 検証 → 改善 を自分で回させる（ループエンジニアリング）」を
スライド作成に当てはめた手順書です。実際に**全140枚のセミナースライドを1セッションで仕上げた往復の型**を、
誰でも再現できる形に落としています。詳しくは [`make-slides/SKILL.md`](make-slides/SKILL.md)。

- **Codex** で使う：`AGENTS.md` を運用ルールに、`SKILL.md` の手順で実行。`/goal` で「はみ出し0・全◯枚」を渡すと自走しやすい。
- **Claude Code** で使う：`make-slides/` を `.claude/skills/make-slides/` に置くか、フォルダごと会話に渡す。

## ② DESIGN.md 50選とは

スライドの「配色・フォント・型」をまとめた正典が50種。各ファイルは独立して使えて、
**貼り付け用の `:root{ … }` ブロック**と**コントラスト規約・禁止表現**を内蔵しています。
ビジネス／親しみ／お金・信頼／ダーク／ミニマル／パステル／ポップの7カテゴリ。一覧は [`design-systems/INDEX.md`](design-systems/INDEX.md)。

---

## この道具が守っていること

- **1スライド1メッセージ** ／ 文章（script.json）と見た目（slides.html）を分ける
- **数値・価格・日付は出典が取れなければ書かない**（確度マーク ✅🔶⚠️）
- **誇張・断定保証をしない**（必ず／絶対／神／ヤバい／裏技 等は使わない）
- **通し番号・枚数は数えて自動採番**（ハードコードしない）
- **コントラスト規約**（差し色は文字に使わない・表見出しは濃地に白字 等）

> このキットは「Codex × ループエンジニアリングで『スライド作成』を半自動化する」の付属品です。
> 解説は親フォルダの [`../README.md`](../README.md) を参照。
